# Bug List

## Fixed
- DOTS calculator returns 154M instead of ~430: meta values (target_squat_kg, target_bench_kg, target_dl_kg, current_body_weight_kg) returned as strings from program_get fission handler due to json.dumps(default=str) on Decimal values. Fixed with recursive _coerce_decimals in program_get/core.py. Also added Number() coercion in DotsCalculator.tsx.
- Competitions page blank: was fission pod_competition cold-start delay. Verified 196 competitions returned correctly now.
- Maxes endpoint returning empty: was fission pod_maxes cold-start delay. Verified targets: squat 220, bench 145, deadlift 250, total 615.
- Analysis page blank/hanging: was originally "fixed" with a 15s AbortController timeout in invokeLambda, but that timeout became the root cause of failures (killed long-running Fission calls like dataset loading). Timeout has been REMOVED — nginx handles request timeouts (5-10 min). Frontend manifest failure is non-fatal - sections poll independently.
- Session design crash (TypeError: item.sets.toFixed is not a function): legacy Session exercises stored sets/reps/kg as strings. Fixed by normalizing exercise numerics at the program_get read boundary (_normalize_session_numerics in program_get/core.py) and making DesignerPage volume math coerce defensively with Number().
- Competition design 502 / Fatigue card empty / Percentile rankings empty: pod_analysis aggregate requirements.txt was missing scipy (individual handlers had it but the image wasn't rebuilt). Added scipy==1.14.1 to pod_analysis/requirements.txt. Image must be rebuilt for the fix to take effect.
- Unauthenticated profile crash (Cannot read properties of undefined (reading 'localeCompare')): profile_get and profile_get_current handlers returned only the inner raw video instead of the full VideoLibraryItem wrapper. Frontend sorter then dereferenced missing session_date/video. Fixed by returning full wrappers from both handlers and making videoSort tolerant of missing dates in both frontend and backend.
- Dashboard upcoming Competition Entries missing: ranking-location settings (country/state) were incorrectly reused to filter the Athlete's own Competition Entries. Fixed by loading the complete canonical Entry list without ranking filters on Dashboard and AnalysisPage, then applying only lifecycle/date filtering locally.
- Weekly Analysis section isolation broken: frontend used Promise.all and one merged data gate, so one failed section hid all others. Fixed with Promise.allSettled — a failed section is recorded as status=error and rendered behind its own boundary; global error only shows if ALL sections fail with no successful payloads.
- Past Block Analysis 502: blockAnalytics.ts called invokeTool('weekly_analysis', ...) which POSTed to a standalone /weekly_analysis endpoint that doesn't exist. Fixed by adding invokeAnalysisTool wrapper in analytics.ts that routes pod_analysis sub-functions (weekly_analysis, correlation_analysis, block_program_evaluation, etc.) through the correct pod_analysis aggregate endpoint with function=<toolName>.

## Known Issues (Fix Forward)
- pod_analysis OOM when loading 782MB OpenPowerlifting CSV. Increased memory limit from 4Gi to 8Gi and changed low_memory=False to low_memory=True. ECR images wiped and CI triggered to rebuild with fix.
- pod_analysis background thread may silently die without setting _df_error. The warm_cache() thread starts at module import but can fail without logging. Need to investigate why the daemon thread dies. Possible cause: unhandled exception in _parse_csvs that doesn't get caught by the except clause.
- ECR DNS resolution intermittent (dial tcp: lookup ... Try again). Pods that need to pull new images may fail. Workaround: delete pod and let it retry until DNS resolves.
- Ranking percentile needs retry polling on frontend. Added 5-attempt retry with 15s interval in Dashboard.tsx.
